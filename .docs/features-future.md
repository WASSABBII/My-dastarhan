## v2 — Мобильное приложение с Push-уведомлениями

### Идея
Всё что сейчас делается через WhatsApp/Telegram — перенести в собственное мобильное приложение.
Гость скачивает RestoBook → получает push-уведомления → нажимает кнопки прямо в приложении.

### Что переезжает из WhatsApp в приложение

| Сценарий | WhatsApp (сейчас) | Приложение (v2) |
|----------|------------------|-----------------|
| Подтверждение брони | WhatsApp сообщение | Push + экран в приложении |
| Напоминание за 1 час | WhatsApp сообщение | Push → открывает бронь |
| ДА / НЕТ / ОТМЕНА | Ответ в WhatsApp | Кнопки в push-уведомлении |
| За 15 мин до конца | WhatsApp сообщение | Push с кнопками |
| Остаюсь / Нас больше / Уходим | Ответ в WhatsApp | Кнопки прямо в уведомлении |
| Запрос отзыва | WhatsApp ссылка | Push → экран оценки в приложении |
| Waitlist — стол освободился | WhatsApp ссылка | Push с кнопкой "Забронировать" |

### Стек

| Компонент | Технология |
|-----------|-----------|
| Мобильное приложение | React Native (iOS + Android) |
| Push-уведомления | Firebase Cloud Messaging (FCM) — бесплатно |
| Кнопки в уведомлениях | FCM Action Buttons (до 3 кнопок) |
| Хранение FCM токена | поле `fcm_token` в таблице `clients` |

### Изменения в БД

```sql
clients
└── fcm_token VARCHAR  ← добавить поле, сохраняем при установке приложения

notification_channel   ← расширить enum:
  "whatsapp" / "telegram" / "app"
```

### Логика выбора канала

```
При отправке уведомления:
  если notification_channel = "app" И fcm_token есть → FCM push
  иначе → WhatsApp / Telegram (как сейчас)
```

WhatsApp остаётся как fallback — для тех кто не установил приложение.

### Экономия

FCM полностью бесплатный → при росте базы клиентов экономия на Twilio значительная:
- 1000 клиентов в приложении = ~$0 вместо ~$50/мес на Twilio

Веб-версия = MVP. Приложение = v2 когда будет база пользователей.
